Description

  1. Recipe Addition:
    • Users can add new recipes through a form.
    • The form includes fields for recipe name, description, and an image.
    • Upon submission, the recipe data is validated and stored in the database.
  2. Recipe Display:
    • All stored recipes are displayed in a table format on a webpage.
    • The table includes columns for recipe ID, name, description, image, and actions.
  3. Image Handling:
    • Recipe images are uploaded through the form and stored in a media directory.
    • Images are displayed in the table along with the other recipe details.
    • Proper URL handling is ensured for displaying images from the media directory.
  4. CRUD Operations:
    • Users can perform Create, Read, Update, and Delete (CRUD) operations on recipes.
    • There are buttons for deleting and updating recipes in the actions column of the table.
    • Deleting a recipe removes it from the database and the table.
    • Updating a recipe allows users to modify its details through a pre-filled form.
